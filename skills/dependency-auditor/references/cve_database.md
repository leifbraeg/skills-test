# CVE Database Reference

## Common Vulnerabilities
- SQL Injection in older ORMs
- XSS in templating engines without auto-escape
- Prototype pollution in utility libraries
- Path traversal in file handlers

## License Types
- MIT: Permissive, minimal restrictions
- Apache 2.0: Permissive with patent protection
- GPL: Strong copyleft
- BSD: Similar to MIT

## Checking Tools
- npm audit
- yarn audit
- Snyk
- OWASP Dependency-Check
- Black Duck

## Version Operators
- `^`: Caret, allows minor/patch updates
- `~`: Tilde, allows patch updates only
- `*`: Any version
- `>=`: Specific minimum version
