# Markdown Documentation Standards

## Structure
- H1 for document title
- H2 for main sections
- H3 for subsections
- Consistent hierarchy (no H1 -> H3 jumps)

## Frontmatter
```yaml
---
name: skill-name
description: One-line description
---
```

## Content Guidelines
- First paragraph is overview
- Code examples for complex concepts
- Links to related resources
- Last update timestamp

## Link Validation
- Internal links work (check with linter)
- External links are current
- No 404 references
- Archive links for stability

## Code Blocks
- Specify language for syntax highlighting
- Keep examples under 30 lines
- Include output examples
- Comment non-obvious sections
